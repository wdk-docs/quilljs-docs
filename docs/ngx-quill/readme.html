ngx-quill ¶

安装¶

  • npm install ngx-quill

  • for projects using Angular < v5.0.0 install npm install ngx-quill@1.6.0

  • install @angular/core, @angular/common, @angular/forms, @angular/platform-browser, quill and rxjs - peer dependencies of ngx-quill

  • include theme stylings: bubble.css, snow.css of quilljs in your index.html, or add them in your css/scss files with @import statements, or add them external stylings in your build process.

对于标准的WebPack, angular-cli 和 tsc 构建¶

  • import QuillModule from ngx-quill:

import { QuillModule } from 'ngx-quill'

  • add QuillModule to the imports of your NgModule:

@NgModule({
  imports: [
    ...,

    QuillModule.forRoot()
  ],
  ...
})
class YourModule { ... }

  • use <quill-editor></quill-editor> in your templates to add a default quill editor

  • do not forget to include quill + theme css in your buildprocess, module or index.html!

  • for builds with angular-cli >=6 only add quilljs to your scripts or scripts section of angular.json, if you need it as a global :)!

HINT: If you are using lazy loading modules, you have to add ``QuillModule.forRoot()`` to your imports in your root module to make sure the ``Config`` services is registered.

对于SystemJS构建(配置)¶

  • 添加鹅毛笔和NGX-鹅毛笔你 paths:

paths: {
  ...
  'ngx-quill': 'node_modules/ngx-quill/bundles/ngx-quill.umd.js',
  'quill': 'node_modules/quill/dist/quill.js'
}

  • 设置格式和依赖于 packages:

packages: {
  'ngx-quill': {
    format: 'cjs',
    meta: {
      deps: ['quill']
    }
  },
  'quill': {
    format: 'cjs'
  }
}

全局配置¶

It is possible to set custom default modules and Quill config options with the import of the QuillModule.forRoot().

@NgModule({
  imports: [
    ...,

    QuillModule.forRoot({
      modules: {
        syntax: true,
        toolbar: [...]
      }
    })
  ],
  ...
})
class YourModule { ... }

If you want to use the syntax module follow the Syntax Highlight Module Guide.

See Quill Configuration for a full list of config options.

The QuillModule exports the defaultModules if you want to extend them :).

QuillEditorComponent¶

暗示¶

Ngx-quill 更新 ngModel 或 formControl 在编辑每一个 user 变化. 检出 QuillJS 源 的 text-change 事件的参数.

如果您在使用编辑器参考直接操纵编辑内容和要更新的模式, 通过 'user' 作为源参数的 QuillJS API 方法.

配置¶

  • ngModel - 设定初始值, 或允许模板驱动形式的双向数据绑定

  • formControl/formControlName - 设定的初始值或允许反应性形式的双向数据绑定

  • readOnly (true | false) 如果用户可以编辑内容

  • formats - 允许的格式/分组的阵列

  • format - 模型格式 - 默认: html, 值: html | object | text | json, 设置模型值类型 - HTML = HTML 字符串, 对象 =quill 操作对象, JSON = quill 操作JSON, text = 纯文本

  • modules - 配置/禁用 quill 模块, e.g工具栏或通过HTML元素默认添加自定义工具栏

{
  toolbar: [
    ['bold', 'italic', 'underline', 'strike'],        // toggled buttons
    ['blockquote', 'code-block'],

    [{ 'header': 1 }, { 'header': 2 }],               // custom button values
    [{ 'list': 'ordered'}, { 'list': 'bullet' }],
    [{ 'script': 'sub'}, { 'script': 'super' }],      // superscript/subscript
    [{ 'indent': '-1'}, { 'indent': '+1' }],          // outdent/indent
    [{ 'direction': 'rtl' }],                         // text direction

    [{ 'size': ['small', false, 'large', 'huge'] }],  // custom dropdown
    [{ 'header': [1, 2, 3, 4, 5, 6, false] }],

    [{ 'color': [] }, { 'background': [] }],          // dropdown with defaults from theme
    [{ 'font': [] }],
    [{ 'align': [] }],

    ['clean'],                                         // remove formatting button

    ['link', 'image', 'video']                         // link and image, video
  ]
};

  • theme - bubble/snow, 默认值是 snow

  • sanitize - 使用角DomSanitizer消毒HTML值 - 默认: true, 布尔 (仅适用于 format=“html”)

  • styles - 设置样式对象, e.g. [styles]="{height: '250px'}"

  • placeholder - 占位符文本, 默认为 Insert text here ...

  • bounds - 编辑, 默认的 document.body 的边界, 通过“自我”的编辑器元件连接

  • maxLength - 对于 maxlength 添加验证 - 设置模式状态 invalid 并添加 ng-invalid 类

  • minLength - 添加验证的使用 minLength - 设置模式状态 invalid 并添加 ng-invalid 类, 只设置无效的, 如果编辑的文本不为空 -> 如果您要检查是否需要文本 -> 使用所需的属性

  • required - 添加验证为必填字段 - [required]="true" - 默认:false, 布尔预期 (无弦!)

  • strict - 默认值: true, 将在严格模式下编辑

  • crollingContainer - 默认的 ‘.ql-editor’, 可以设置滚动容器

  • 使用自定义选项, 添加例如自定义字体大小 -> 这个覆盖此选项 globally!

  • 可能性来创建经由投影槽 [quill-editor-toolbar] 自定义工具栏:

<quill-editor>
  <div quill-editor-toolbar>
    <span class="ql-formats">
      <button class="ql-bold" [title]="'Bold'"></button>
    </span>
    <span class="ql-formats">
      <select class="ql-align" [title]="'Aligment'">
        <option selected></option>
        <option value="center"></option>
        <option value="right"></option>
        <option value="justify"></option>
      </select>
      <select class="ql-align" [title]="'Aligment2'">
        <option selected></option>
        <option value="center"></option>
        <option value="right"></option>
        <option value="justify"></option>
      </select>
    </span>
  </div>
</quill-editor>

  • customToolbarPosition - 如果您正在使用自定义工具栏工作, 就可以切换位置:). - 默认: top, 可能值 top, bottom

  • debug - 设置日志级别 warn, error, log 要么 false 停用记录, 默认: warn

  • trackChanges - 检查是否只有 user (翮源用户)或 all 变化应触发模型更新, 默认情况下 user. 使用 all 不推荐, 它会导致一些意想不到的副作用.

  • preserveWhitespace - 默认: false - possbility使用预标记, 而不是一个div标签的 contenteditable 区域保留重复空格 | 如果谨慎使用语法插件 相关的问题 使用

  • classes - CSS类的空格分隔列表, 它会被添加到编辑元素

输出¶

  • onEditorCreated - 编辑器实例

  • 使用此输出来获取编辑器实例, 并直接使用它. 在此之后输出已调用组件是稳定的, 所有的听众都绑定

editor // Quill

  • onContentChanged - 文字更新

{
  editor: editorInstance, // Quill
  html: html, // html string
  text: text, // plain text string
  content: content, // Content - operatins representation
  delta: delta, // Delta
  oldDelta: oldDelta, // Delta
  source: source // ('user', 'api', 'silent' , undefined)
}

  • onSelectionChanged - 选择更新, 也引发了的onblur和聚焦状态, 因为选择改变

{
  editor: editorInstance, // Quill
  range: range, // Range
  oldRange: oldRange, // Range
  source: source // ('user', 'api', 'silent' , undefined)
}

  • onEditorChanged - 文本或选择更新 - 独立于源的

{
  editor: editorInstance, // Quill
  event: 'text-change' // event type
  html: html, // html string
  text: text, // plain text string
  content: content, // Content - operatins representation
  delta: delta, // Delta
  oldDelta: oldDelta, // Delta
  source: source // ('user', 'api', 'silent' , undefined)
}

要么

{
  editor: editorInstance, // Quill
  event: 'selection-change' // event type
  range: range, // Range
  oldRange: oldRange, // Range
  source: source // ('user', 'api', 'silent' , undefined)
}

  • onFocus - 编辑聚焦

{
  editor: editorInstance, // Quill
  source: source // ('user', 'api', 'silent' , undefined)
}

  • onBlur - 编辑模糊

{
  editor: editorInstance, // Quill
  source: source // ('user', 'api', 'silent' , undefined)
}

QuillViewComponent, QuillViewHTMLComponent 及如何呈现编辑内容¶

在大多数情况下, 一个所见即所得的编辑器在后台办公用于存储内容数据库。在另一侧上这个值应该被使用, 以显示该内容到最终用户。

在大多数情况下, html 格式被使用, 但它不是由QuillJS推荐的, 因为它有意向的是固体, 易于维护的编辑器。正因为如此, 它使用印迹和内容和操作的对象表示。

此内容对象是易于储存和维护, 因为没有HTML语法分析必要的。所以, 你即使切换到另一个编辑器是很容易的, 当你可以与合作。

NGX-quill 提供了一些辅助组件, 以呈现quilljs内容.

QuillViewComponent - 使用QuillJS呈现内容¶

一般来说QuillJS建议使用QuillJS实例来展示您的内容。只需创建一个没有工具栏, 并以只读模式鹅毛笔编辑器。通过一些简单的CSS行, 你可以删除内容周围的默认边框。

作为帮助 NGX-quill 提供了一个组件, 你可以通过像模块, 格式, 格式, customOptions的 鹅毛笔, editor 的许多选择, 但仅呈现内容为只读和不带工具栏。进口是 content 输入, 在那里你可以传给你想呈现的内容编辑。

配置¶

  • content - 要呈现的内容

  • formats - 允许的格式/分组的阵列

  • format - 模型格式 - 默认: html, 值: html | object | text | json, 设置模型值类型 - HTML = HTML 字符串, 对象 =quill 操作对象, JSON = quill 操作JSON, text = 纯文本

  • modules - 配置/禁用套筒模块

  • theme - bubble/snow, 默认值是 snow

  • debug - 设置日志级别 warn, error, log 要么 false 停用记录, 默认: warn

  • 使用自定义选项, 添加例如自定义字体大小 -> 这个覆盖此选项 globally!

  • strict - 默认值: true, 将在严格模式下编辑

  • preserveWhitespace - 默认: false - possbility使用预标记, 而不是一个div标签的 contenteditable 区域保留重复空格 | 如果谨慎使用语法插件 相关的问题 使用

<quill-view [content]="content" format="text" theme="snow"></quill-view>

QuillViewHTMLComponent - 运用 angular [innerHTML]¶

Most of you will use the html format (even it is not recommended). To render custom html with angular you should use the [innerHTML] attribute.

但也有一些陷阱:

  1. You need to have the quill css files loaded, when using classes and not inline styling (https://quilljs.com/guides/how-to-customize-quill/#class-vs-inline)

  2. When using classes use a div-tag that has the innerHTML attribute and add the ql-editor class. Wrap your div in another div-tag with css classes ql-container and your theme, e.g. ql-snow.:

<div class="ql-container ql-snow" style="border-width: 0;">
  <div class="ql-editor" [innerHTML]="byPassedHTMLString">
  </div>
</div>

  1. Angular has html sanitation, so it will strip unkown or not trusted parts of your HTML - just mark your html as trusted (DomSanitizer)

这之后你的内容应该像你所期望的.

If you store html in your database, checkout your backend code, sometimes backends are stripping unwanted tags as well ;).

As a helper ngx-quill provides a component where you can simply pass your html string and the component does everything for you to render it:

  • 添加必要的CSS类

  • 旁路HTML卫生

<quill-view-html [content]="htmlstring" theme="snow"></quill-view-html>

配置¶

As inputs you can set the content and optional the theme (default is snow).

  • content - HTML字符串呈现

  • theme - bubble/snow, 默认值是 snow

安全提示¶

Angular 模板提供针对XSS一些保证在所有输入客户端消毒的形式 https://angular.io/guide/security#xss.

NGX- quill 提供的配置放慢参数 sanitize 消毒的HTML字符串作为 ngModel 或 formControl 到组件通过.

据 每默认禁用 避免剥离的内容或风格, 这是不预期.

但 推荐 激活这个选项, 如果您正在使用HTML字符串作为模型值工作.
